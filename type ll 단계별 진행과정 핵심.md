1. 문제 정의
이유: 분류(Classification)인가, 회귀(Regression)인가?

가장 먼저 내가 풀어야 할 문제의 성격을 파악해야 그에 맞는 모델과 평가 지표(ROC-AUC, RMSE 등)를 선택할 수 있다.

# 2. 라이브러리 및 데이터 불러오기
이유: 분석 도구 세팅

pandas, sklearn 등 필요한 도구를 가져오고, 흩어져 있는 train, test 데이터를 메모리에 올려야 분석을 시작할 수 있다.

# 3. 탐색적 데이터 분석 (EDA)
이유: 데이터의 민낯(결측치, 이상치, 분포) 확인

데이터가 깨끗한지, 타겟 값(Outcome)이 한쪽으로 쏠려 있지는 않은지 알아야 성능을 높일 **'전처리 전략'**을 세울 수 있다.

# 4. 데이터 전처리
이유: 기계가 학습 가능한 형태로 가공

컴퓨터는 '문자'를 이해하지 못하므로 숫자로 바꾸고(인코딩), 비어있는 값(결측치)을 채워 모델이 오류 없이 학습할 수 있게 만든다.

# 5. 검증 데이터 분할
이유: 모델의 실전 감각 테스트

학습용 데이터 중 일부를 따로 떼어 '모의고사'를 치러봄으로써, 모델이 새 데이터(Test)에서도 잘 작동할지 미리 예측해 보는 과정이다.

# 6. 머신러닝 학습 및 평가
이유: 최적의 해결책(패턴) 도출

데이터 속의 규칙을 찾아내는 '공부'를 시키고, 모의고사 점수(평가 지표)를 확인하여 이 모델을 그대로 제출해도 될지 결정한다.

# 7. 예측 및 결과 파일 생성
이유: 최종 답안지 작성

완성된 모델을 실제 test 데이터에 적용하여 예측값을 뽑아내고, 시험 시스템이 요구하는 형식(.csv)으로 저장하여 제출을 완료한다.
